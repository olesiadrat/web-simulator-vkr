# План разработки

## Этап 1. Каркас проекта

- Создать структуру `backend/`, `frontend/`, `docs/`.
- Подготовить README и `.gitignore`.
- Подготовить Django-проект и зависимости backend.
- Описать переменные окружения.

## Этап 2. Backend API

- Модели: `Scenario`, `Session`, `BugReport`, `ReferenceBug`.
- Serializers для REST API.
- ViewSet endpoints для сценариев, сессий и баг-репортов.
- Seed-команда для демонстрационного сценария.

## Этап 3. Frontend

- React + TypeScript + Vite.
- Страницы: список сценариев, тренажер.
- API-клиент.
- Создание сессии при старте сценария.

## Этап 4. JSON Renderer

- Поддержка элементов: `text`, `input`, `button`, `link`, `card`, `image`.
- Переходы между страницами через action `navigate`.
- Хранение текущей страницы и введенных значений.

## Этап 5. Баг-репорты

- Форма создания баг-репорта.
- Список багов текущей сессии.
- Сохранение через backend API.

## Этап 6. Демонстрация для ВКР

- Демонстрационный сценарий по макету сервиса отдыха.
- README с командами запуска.
- Описание архитектуры и API.
- Скриншоты интерфейса.
